An Uber Shader specialised for rendering Sprites in Unity. Even though it\textquotesingle{}s designed for Sprites it can be used for a whole range of uses. It supports a wide range of optional shader features that won\textquotesingle{}t effect performance unless they are used. It also supports per-\/pixel effects such as normal maps and diffuse ramping whilst using Vertex Lit rendering.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md132}{}\doxysubsubsection{\texorpdfstring{Lighting}{Lighting}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md132}
The shaders support lighting using both Forward Rendering and Vertex Lit Rendering. Forward rendering is more accurate but is slower and crucially means the sprite has to write to depth using alpha clipping to avoid overdraw. Vertex lit means all lighting can be done in one pass meaning full alpha can be used.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md133}{}\doxysubsubsection{\texorpdfstring{Normal Mapping}{Normal Mapping}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md133}
Normals maps are supported in both lighting modes (in Vertex Lit rendering data for normal mapping is packed into texture channels and then processed per pixel).\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md134}{}\doxysubsubsection{\texorpdfstring{Blend Modes}{Blend Modes}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md134}
Easily switch between blend modes including pre-\/multiplied alpha, additive, multiply etc.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md135}{}\doxysubsubsection{\texorpdfstring{Rim Lighting}{Rim Lighting}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md135}
Camera-\/space rim lighting is supported in both lighting modes.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md136}{}\doxysubsubsection{\texorpdfstring{Diffuse Ramp}{Diffuse Ramp}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md136}
A ramp texture is optionally supported for toon shading effects.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md137}{}\doxysubsubsection{\texorpdfstring{Shadows}{Shadows}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md137}
Shadows are supported using alpha clipping.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md138}{}\doxysubsubsection{\texorpdfstring{Gradient based Ambient lighting}{Gradient based Ambient lighting}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md138}
Both lighting modes support using a gradient for ambient light. In Vertex Lit mode the Spherical Harmonics is approximated from the ground, equator and sky colors.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md139}{}\doxysubsubsection{\texorpdfstring{Emission Map}{Emission Map}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md139}
An optional emission map is supported.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md140}{}\doxysubsubsection{\texorpdfstring{Camera Space Normals}{Camera Space Normals}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md140}
As sprites are 2d their normals will always be constant. The shaders allow you to define a fixed normal in camera space rather than pass through mesh normals. This not only saves vertex throughput but means lighting looks less \textquotesingle{}flat\textquotesingle{} for rendering sprites with a perspective camera.\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md141}{}\doxysubsubsection{\texorpdfstring{Color Adjustment}{Color Adjustment}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md141}
The shaders allow optional adjustment of hue / saturation and brightness as well as applying a solid color overlay effect for flashing a sprite to a solid color (eg. for damage effects).\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md142}{}\doxysubsubsection{\texorpdfstring{Fog}{Fog}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md142}
Fog is optionally supported\hypertarget{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md143}{}\doxysubsection{\texorpdfstring{To Use}{To Use}}\label{md__d_1_2_unity_2_senior_olympics_2_assets_2_spine_2_runtime_2spine-unity_2_shaders_2_sprite_2_r_e_a_d_m_e_autotoc_md143}
