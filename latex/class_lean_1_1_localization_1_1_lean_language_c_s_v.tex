By default they should be in the format\+: Phrase Name Here = Translation Here // Optional Comment Here NOTE\+: This component only handles loading one CSV file from one language. If you have multiple languages, then you must make multiple CSV files for each, and a matching {\bfseries{\doxylink{class_lean_1_1_localization_1_1_lean_language_c_s_v}{Lean\+Language\+CSV}}} component to load each.

{\footnotesize\ttfamily \label{class_lean_1_1_localization_1_1_lean_language_c_s_v_a4173f82c6a95155ed1aeaa2d9bfece39} 
\mbox{\hyperlink{class_lean_1_1_localization_1_1_lean_language_c_s_v_acec7cffc936b7c410a909411a86d9a33}{Cache\+Type}} Lean.\+Localization.\+Lean\+Language\+CSV.\+Cache}



This allows you to control when the CSV file is loaded or unloaded. The lower down you set this, the lower your app\textquotesingle{}s memory usage will be. However, setting it too low means you can miss translations if you haven\textquotesingle{}t translated absolutely every phrase in every language, so I recommend you use {\bfseries{Load\+Immediately}} unless you have LOTS of translations. Load\+Immediately = Regardless of the language, the CSV will load when this component activates, and then it will be kept in memory until this component is destroyed. Lazy\+Load = The CSV file will only load when the {\bfseries{Current\+Language}} or {\bfseries{Default\+Language}} matches the CSV language, and then it will be kept in memory until this component is destroyed. Lazy\+Load\+And\+Unload = Like {\bfseries{Lazy\+Load}}, but translations will be unloaded if the {\bfseries{Current\+Language}} or {\bfseries{Default\+Language}} differs from the CSV language. Lazy\+Load\+And\+Unload\+Primary\+Only = Like {\bfseries{Lazy\+Load\+And\+Unload}}, but only the {\bfseries{Current\+Language}} will be used, the {\bfseries{Default\+Language}} will be ignored.
