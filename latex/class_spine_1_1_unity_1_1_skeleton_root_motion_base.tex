This callback can be used to apply root-\/motion in a custom way. It is raised in Fixed\+Update after (when \doxylink{class_spine_1_1_unity_1_1_skeleton_root_motion_base_afc74579bf1beb4419b3475d917c4cec5}{disable\+On\+Override} is set to false) or instead of when root-\/motion would be applied at the Rigidbody. When \doxylink{class_spine_1_1_unity_1_1_skeleton_animation_a63be211aa9bb50fcd5aba2230bb90450}{Skeleton\+Animation.\+Update\+Timing} is set to \doxylink{namespace_spine_1_1_unity_ad8a24b651177d0df59c5638263ab76dca17f892d49682dd26b9df3619a5d0e5f9}{Update\+Timing.\+In\+Update}, multiple animation frames might take place before before {\ttfamily Fixed\+Update} is called once. The callback parameters {\ttfamily translation} and {\ttfamily rotation} are filled out with the (cumulated) skeleton-\/space root-\/motion since the the last {\ttfamily Fixed\+Update} call. You can use e.\+g. {\ttfamily transform.\+Transform\+Vector()} to transform skeleton-\/space root-\/motion to world space.
